# 知体知医 - 个性化健康管理Web应用

## 项目概述

**知体知医**是一款基于Web的个性化健康管理应用，旨在帮助用户建立健康意识、管理个人健康数据、获取疾病知识，并通过自测评估健康风险。应用采用移动端优先的设计理念，提供简洁直观的用户界面和完整的健康管理功能。

## 核心功能

### 1. 综合健康分系统
- 基于五大维度计算健康分数：症状反馈、风险评估、档案完整度、打卡记录、AI评估
- 动态SVG圆环进度条可视化展示
- 历史趋势图表追踪健康变化

### 2. 反馈记录模块
- 快捷标签选择（很好/一般/不舒服）
- 详细文字描述
- 检查报告/照片上传
- 历史反馈记录查看

### 3. 风险自测模块
- 根据用户疾病史智能推荐相关测试项目
- 50+项专业自测项目，涵盖心血管、内分泌、消化、呼吸、神经等多个系统
- 每项测试包含：检测方法、正常标准、异常提示
- **智能就医建议**：异常结果自动提示推荐科室、建议检查项目、紧急程度

### 4. 提分学习模块
- **疾病知识库**：40+种常见疾病的详细知识
  - 发病原因、发病机理、诊断标准
  - 西医治疗方案、中医证型分类
  - 推荐科室、建议检查项目
- **个性化推荐**：仅根据用户添加的疾病推荐学习内容
- **生活建议卡片**：心理调养、饮食建议（宜/忌）、运动处方、环境调节、睡眠管理
- **计划管理**：可将建议加入个人计划，完成自动打卡

### 5. 健康档案模块
- 基本信息管理（昵称、年龄、性别、血型）
- 疾病史记录
- 过敏史分类管理（药物/食物/环境）
- 健康数据导出

## 技术架构

### 技术栈
| 技术 | 说明 |
|------|------|
| HTML5 | 页面结构 |
| TailwindCSS (CDN) | 样式框架 |
| Vanilla JavaScript | 交互逻辑 |
| localStorage | 数据持久化 |
| Canvas API | 趋势图表绘制 |

### 架构特点
- **单页应用（SPA）**：通过JavaScript实现页面切换，无刷新体验
- **移动端优先**：最大宽度430px，适配手机屏幕
- **离线可用**：数据存储在本地localStorage，无需服务器
- **零依赖**：除TailwindCSS CDN外，不依赖任何第三方库

### 数据结构
```
localStorage (前缀: ztzy_)
├── profile          # 用户基本信息
├── diseases         # 疾病史列表
├── allergies        # 过敏史
├── feedbacks        # 反馈记录
├── test_result      # 自测结果
├── my_plan          # 个人计划
├── today_completed  # 今日完成记录
├── checkins         # 打卡记录
├── score_history    # 健康分历史
└── share_code       # 分享码
```

## 数据资源

### 疾病知识库（40+种疾病）
覆盖多个医学领域：
- **心血管系统**：高血压、心脏病、高血脂、冠心病、心律失常、心肌炎
- **内分泌系统**：糖尿病、甲亢、甲减、痛风、肥胖症
- **消化系统**：胃炎、脂肪肝、胃溃疡、肠易激综合征、胆囊炎、胰腺炎
- **呼吸系统**：哮喘、慢性支气管炎、过敏性鼻炎、肺炎
- **神经系统**：失眠、偏头痛、癫痫、帕金森病
- **骨骼肌肉系统**：颈椎病、腰椎间盘突出、骨质疏松、类风湿关节炎
- **精神心理**：焦虑症、抑郁症、强迫症
- **泌尿系统**：慢性肾病、肾结石、前列腺炎
- **皮肤科**：湿疹、银屑病、痤疮
- **妇科**：月经不调、多囊卵巢综合征
- **其他**：贫血、干眼症、甲状腺结节、高尿酸血症

### 测试项目库（50+项测试）
每项测试包含完整信息：
- 测试ID、名称、描述
- 详细检测方法
- 正常标准参考值
- 异常提示及就医建议

### 就医建议库
为每种疾病配置：
- 推荐就诊科室
- 建议检查项目
- 紧急程度分级（急诊/尽快就诊/需及时就诊/门诊随访）

## 经验总结

### 1. AI辅助开发的优势

**效率提升**
- 传统开发需要数天的功能，AI可在分钟内完成
- 无需手动编写重复性代码（如40+种疾病的数据结构）
- 自动处理部署流程

**质量保障**
- 代码结构规范，遵循最佳实践
- 自动添加必要的注释和错误处理
- 响应式设计自动适配

**知识整合**
- AI能够整合医学知识库内容
- 自动建立疾病-测试-就医建议的关联
- 生成专业的Excel数据模板

### 2. 人机协作模式

**人类角色**
- 明确业务需求和目标用户
- 提供医学专业知识指导
- 验证功能是否符合预期
- 决定迭代方向

**AI角色**
- 将需求转化为技术实现
- 生成代码和数据
- 处理部署和配置
- 快速响应修改请求

### 3. 开发最佳实践

**需求表达**
- 清晰描述功能目标和约束条件
- 提供具体的数据结构和交互细节
- 分阶段提出需求，逐步迭代

**迭代策略**
- 先实现核心功能，再扩展内容
- 每次迭代聚焦一个明确目标
- 及时验证和反馈

**技术选择**
- 选择适合项目规模的技术栈
- 优先使用成熟的CDN资源
- 保持代码简洁，避免过度工程

### 4. 医疗健康类应用的注意事项

**数据安全**
- 本项目采用本地存储，数据不上传服务器
- 用户隐私得到保护
- 适合作为个人健康管理工具

**专业性**
- 内容需基于权威医学资料
- 明确提示"仅供参考，不替代专业医疗建议"
- 异常结果引导用户就医

**用户体验**
- 简化操作流程
- 提供清晰的指引和反馈
- 移动端优先设计
